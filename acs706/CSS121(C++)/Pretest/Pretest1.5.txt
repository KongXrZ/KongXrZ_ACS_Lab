วิเคราะห์ Time Complexity (Big-O Notation)
1. การสร้างลิสต์ (Insert)
Big-O: O(n)
ประเภท: เชิงเส้น (Linear Time)
คำอธิบาย: เกิดจากการที่เราต้องวนลูปสร้างนักศึกษาทีละคน ตั้งแต่คนแรกจนถึงคนที่ $n$ (ในที่นี้คือ 25) แต่ละรอบของลูปใช้เวลาคงที่ในการสุ่มค่าและกำหนด ID ดังนั้นเวลาที่ใช้จะเพิ่มขึ้นเป็นเส้นตรงตามจำนวนนักศึกษาที่เพิ่มขึ้น

2. การจัดเรียง (Sort) - Selection Sort
Big-O: O(n^2)
ประเภท: กำลังสอง (Quadratic Time)
คำอธิบาย: เกิดจากการใช้ Nested Loop (ลูปซ้อนลูป) โดยลูปนอกวิ่ง n ครั้ง และลูปในวิ่งเพื่อหาค่าที่น้อยที่สุดอีกประมาณ n ครั้ง ทำให้เกิดการเปรียบเทียบข้อมูลทั้งหมดประมาณ n x n รอบ 
แม้จะมีการ swap เพียงรอบละครั้ง แต่ในเชิงทฤษฎีการเปรียบเทียบยังเป็น O(n^2) 

3. การค้นหา (Search) และ การท่องโหนด (Traversal)
Binary Search: O(log n) — เชิงลอการิทึม (Logarithmic Time)
คำอธิบาย: เกิดจากการแบ่งครึ่งข้อมูลที่ค้นหาในทุกๆ ขั้นตอน (Divide and Conquer) ทำให้จำนวนข้อมูลที่ต้องพิจารณาลดลงอย่างรวดเร็ว เหมาะสำหรับการค้นหาในข้อมูลที่เรียงลำดับแล้ว
In-order Traversal: O(n) — เชิงเส้น (Linear Time)
คำอธิบาย: การแสดงผลนักศึกษาทุกคนจาก Tree ต้องแวะไปเยี่ยม (Visit) ทุกโหนด โหนดละ 1 ครั้ง ดังนั้นเวลาที่ใช้จะขึ้นอยู่กับจำนวนโหนดทั้งหมด (n)

4. การหาค่าสถิติ (Mean, Mode, Median)
Mean (ค่าเฉลี่ย): O(n) — เชิงเส้น (Linear Time)
คำอธิบาย: ต้องวนลูปเพื่อเข้าถึงคะแนนของทุกคน 1 รอบ เพื่อนำมาหาผลรวมก่อนหารด้วย n
Mode (ฐานนิยม): O(n) — เชิงเส้น (Linear Time)
คำอธิบาย: เราใช้เทคนิค Frequency Table (ถังนับความถี่) โดยวนลูปอ่านคะแนนนักศึกษาทุกคนเพียงรอบเดียวเพื่อบันทึกลงถัง และวนลูปที่ความยาวคงที่ (101 ช่อง) เพื่อหาตัวที่ซ้ำสุด
Median (มัธยฐาน): O(n log n) — เชิงเชิงเส้น-ลอการิทึม (Linearithmic Time)
คำอธิบาย: หัวใจสำคัญคือข้อมูลต้องถูก Sort ก่อนถึงจะหาค่ากลางได้ ซึ่งอัลกอริทึม std::sort ที่เราใช้มีประสิทธิภาพเป็น O(n log n)
